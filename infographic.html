Context Resolution — how ancestor contexts are merged
Input path
e.g. /repo/services/api/foo.py
›
Normalize & validate
Must be inside a configured root; reject traversal
›
Find ancestors
All projects whose root is a prefix of the path
›
Sort by depth
Shallowest (broadest) first
›
Merge sections
Per-field strategy (see table)
›
EffectiveContext
Merged fields + raw combined markdown + resolution trace
⚖️ Section Merge Rules
Section(s) Strategy Behaviour
purpose, overview nearest wins Deepest non-empty value replaces all parents
stack, commands, setup, constraints, architecture, notes, testing combine + dedup All items merged; order preserved; duplicates removed
coding_rules, definition_of_done keyed merge Narrower scope overwrites broader scope per key
tags combined + dedup All tags from all ancestors, deduplicated
🌲 Hierarchy Example
/repo/                     ← AGENTS.md  (depth 1)
  services/                ← AGENTS.md  (depth 2)
    api/                   ← AGENTS.md  (depth 3)
      foo.py               ← target path

Resolution trace for foo.py:
  1. /repo/AGENTS.md       (broadest)
  2. /repo/services/AGENTS.md
  3. /repo/services/api/AGENTS.md  (narrowest → wins scalars)

purpose: taken from depth-3 file if present
commands: union of all three files (deduped)
coding_rules: deeper keys overwrite shallower

Safety & Performance Features
Path Traversal Prevention All paths normalized and validated to stay inside configured roots.
SHA256 Concurrency Lock Managed file writes accept an expected_sha256; mismatches are rejected (optimistic lock).
Atomic File Writes All writes go to a temp file first, then renamed — no partial writes.
Parse Caching Files unchanged since last scan are read from cache, not re-parsed — fast refreshes.
Referential Integrity Installations must reference valid account_id and device_id; orphan prevention on delete.
Symlink Safety follow_symlinks defaults to false; opt-in only to prevent escaping roots.
