      Help & Reference

      How everything works

      Agents Registry discovers and indexes the hierarchical AGENTS.md / agents.md files scattered across your configured root directories, then resolves the effective, inherited instructions for any path in your codebase. The same data is available here in the admin UI and to agents over the MCP server.

      Core concepts

      ▸ AGENTS.md & CLAUDE.md files

      Plain-Markdown files that hold instructions for AI agents — purpose, tech stack, commands, setup steps, constraints, coding rules, and a definition of done. Drop one at any level of your tree; deeper files describe more specific scopes.

      ▸ Hierarchy merge (nearest wins)

      For a given path the registry collects every ancestor file (shallowest first) and merges them. Nearest wins for purpose and overview; lists like commands, constraints, setup and stack are combined and de-duplicated; keyed sections such as coding_rules are replaced entry-by-entry by the nearest file.

      ▸ The MCP server surface

      Agents connect over MCP and get 5 tools (list_projects, get_project, resolve_context, search_projects, refresh_index), 5 resources under agents://, and 2 prompt templates. This admin UI calls the same underlying registry.

      ▸ The cache & refresh

      Parsed files are cached on disk and keyed by path. A Refresh Index only re-parses files whose modification time, size, or content hash changed — so refreshing is cheap and safe to run often.

      The panels, explained

      01 Registry
      Lists every indexed project and lets you search across names, paths, tags and file contents (all terms must match). Use ↻ Refresh Index after editing files on disk, and Reload Projects to repaint the list. Click a project to load it into the Inspector.
      02 Inspector
      Resolves the effective inherited context. Paste any absolute path under a configured root and press → Resolve Context to see the merged result, or use Project Effective Context for a selected project's root. Paths outside the configured roots are rejected.
      03 Accounts & Devices
      Your inventory of who (accounts) and what (devices) runs agents. Load Inventory lists them; Coverage Report summarizes gaps. The two cards create a new account or device.
      04 Installations & Skills
      Records which agent (and which skills) is installed on a given account + device. Assign Installation creates the link; Where is this agent? and Skills on account/device answer locate-style questions across the inventory.
      05 Markdown Editor
      Read and edit the managed AGENTS.md / CLAUDE.md files directly. List Managed Files finds them, Read File loads one, Preview Save shows the diff, and ✓ Commit Save writes it back (with a hash check to avoid clobbering concurrent edits).
      06 Review Queue
      Pending memory/registry-update proposals raised by agents. Edit the heading or content inline, then approve (applies it to the target file) or reject it. Toggle the checkbox to see approved and rejected history.
